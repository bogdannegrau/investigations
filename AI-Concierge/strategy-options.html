Integration Strategy — Three Options
● Option 1 — Current

LivePerson

Continue with existing LP setup. AI via LP's built-in integrations (IBM Watson, AWS Lex). Human agent support already operational.

  • ✓ Human agent support (live today)
  • ✓ AI bots via Watson / Lex
  • ✗ Limited AI model choice
  • ✗ No control over AI behaviour
  • ~ Vendor lock-in risk
  • ✓ Minimal build effort
Build effort Low
Flexibility Low
Ongoing cost High (LP licence)
Time to value Immediate
★ Option 2 — In-House

CCC Handler

Build our own Customer Concierge Controller. Plug in any AI provider. Escalate to LP for human agents, or build agent tooling in-house.

  • ✓ Full AI model flexibility
  • ✓ Custom guardrails & cost control
  • ✓ MCP integrations (HLP, FBR, CPP)
  • ✓ A/B test multiple LLMs
  • ~ Human agents via LP or DIY
  • ~ Higher initial build effort
Build effort Medium–High
Flexibility High
Ongoing cost Low (pay-per-use)
Time to value ~3 months (AI mode)
◆ Option 3 — Alternative

New Vendor

Replace LP with a modern platform offering both AI orchestration and human agent support. E.g. Intercom, Zendesk AI, Ada, Kustomer.

  • ✓ Human agent support included
  • ✓ Built-in AI capabilities
  • ~ Some AI model flexibility
  • ✗ Migration effort from LP
  • ~ Vendor lock-in (new vendor)
  • ~ Customisation limits vary
Build effort Medium (migration)
Flexibility Medium
Ongoing cost Medium (licence)
Time to value Depends on vendor
